# 原理

冒泡排序是一种**交换排序**，思想是：
- 一直去遍历没有排序的部分，如果发现前一个和后一个的顺序错误，就将他们进行交换  
- 每一轮排序，都会将没排序部分的最大元素冒泡到末尾，直到整个数组排序完成

# 优点  
- 简单
- 稳定：相同的元素之间不进行交换，  
稳定指的是相同元素排序后的相对位置不发生改变，在多字段的排序下：想象以下场景，要求现在对入职时间进行排序，下来再要求对年龄进行排序，如果此时使用不稳定的排序方式，那么就会打乱之前按入职时间的排序
- 原地排序：空间复杂度是O(1),只在原来的数组上进行操作，节省内存
- 优化版对接近有序的数组高效，如果发现这躺没有交换，那证明已经有序了

# 缺点
- 效率太低：时间复杂度是O(n^2)
- 比较和交换次数多：虽然改进版本，但是不治本
- 不能处理大规模数据:数据一大，性能直线下滑，只时候处理小规模的数据，go开发中几乎不用，一般用sort包
