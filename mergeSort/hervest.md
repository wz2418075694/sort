# 原理
- 归并排序的思想是将一个乱序的数组折半递归拆分，直到数组中只有一个元素(一个元素本身是有序的)，从最小的数组开始，用归并的方法两两合并成新的有序数组，直到合并成完整的有序数组
- 归并排序=拆分+合并

# 优点
- 效率高：空间复杂度：O(nlogn)，即使数组完全逆序，效率也不受影响
- 稳定排序：归并的条件是：arr[i] <= arr[j]先取左边的数组中的元素放入要合并的数组中，相对位置没有发生改变  
适合多字段排序  
3(1) 2(2) 3(3) -> 2(2) 3(1) 3(3)
- 适合大规模数据：百万级数据也能高效处理
可并行化：拆分后的子数组可以并行排序，适合分布式场景

# 缺点
- 空间开销大：需要O(n)的空间，内存受限的场景不适用
- 不适合小规模数据：n<100的时候不如插入排序

# 避坑
- 计算mid不能用mid=(left+right)/2,要用mid=left+(right-left)/2，避免溢出
- 合并用arr[i]<=arr[j],不能用arr[i]<arr[j],不然丢失稳定性
- 拷贝回原数组中时，arr[gap+p]
- 漏写边界处理 len(arr)<=1

# 面试
### 归并排序的核心思想是什么，它为什么是稳定的？
- 核心思想：分治思想，先递归拆分数组为单元素，然后在合并有序的子数组，直到合并成完整的数组
- 合并时用的arr[i]<=arr[j]，这就导致有相等的元素时，优先取左边的子数组的元素，保证的相同元素的相对位置不变

### 能手写出完整的代码，并说出关键的优化点
- mid=left+(right-left)/2避免（left+right）/2 溢出
- arr[i] <= arr[j] 保证稳定性
- len(arr) <= 1 提前返回，避免无效递归

### 归并排序和快速排序怎么选
- 要求稳定性+大规模数据->归并排序
- 无稳定性需求+内存受限+追求极致的性能->快速排序
- go中的sort包中用的是：快速排序+堆排序+插入排序的混合实现，比手写归并、快速更健壮

### 归并排序的空间复杂度为什么是O(n)，能否优化？
- 在排序中需要最多n个长度的临时数组保存数据
- 可优化为原地归并排序，但是逻辑复杂，面试不用手动写
- 可以用非递归，减少递归站的开销，但是无法避免空间O(n)的开销
