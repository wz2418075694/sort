# 希尔排序
- 首先我们知道插入排序的痛点是：如果这个数组是完全逆序的，那么所有的元素都要向前移动，这导致此时的效率极低，所以就诞生了希尔排序
- 原理：希尔排序其实是插入排序的优化版本，是将数组先分组，此时数据就变小了，将每组的数据先用插入排序排好，让数据整体接近有序，然后逐渐减小步长，直到步长=1，如此循环以上的过程，然后在用简单插入排序，此时数组已经基本有序，再排就很快
- 综上所述：希尔排序=大间隔的插入排序+步长递减+普通插入排序

# 优点
- 效率远高于插入排序：平均的时间复杂度：O(n^1.3),最坏O(n^2)
- 原地排序：空间复杂度是O(1)
- 适合处理中等规模(千级-万级)乱序数组

# 缺点
- 不稳定；分组的时候相同的元素可能被分到了不同的组，跨组交换后相对位置可能会改变
- 步长的选择直接影响排序的效率，

# 面试题

### 希尔排序为什么比插入排序快？
- 插入排序的痛点：当数组逆序的时候，所有的元素只能一步步相邻移动
- 希尔排序将数组分组后，可以让元素跨步长的大距离移动，从而减少了插入排序移动的次数，当数据基本有序时，再用插入排序收尾

### 希尔排序稳定吗？
- 不稳定
- 分组的时候相同元素可能被分到不同的组，跨组交换的时候相对位置发生了改变
 
### 希尔排序的核心是什么
- 步长分组+插入排序
