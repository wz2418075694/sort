# 选择排序原理
- 找最值，放到位  
将数组分为已排序区和未排序区，每次找未排序区的最值和未排序区的第一个元素进行交换，直到排序完成

# 优点
- 简单
- 交换次数少：不同于冒泡排序，适用于交换成本高的场景(大结构体)，每次只将最值和未排序区的第一个元素交换
- 原地排序：空间复杂度是O(1),不需要额外的空间

# 缺点
- 效率低：时间复杂度任何情况下都是是O(n^2)，即便数组已经有序，还是要去找最值
- 不稳定：选择排序是将找到的最值和未排序区的第一个元素进行交换，这就导致了可能他们之间跨越了多个元素，导致具有相同值的元素的相对位置发生变换  
面试举必须举例说明： 3(0) 2(1) 3(2) 1(4) -> 1(2) 2(1) 3(2) 3(0)
- 对数据敏感度低：如果这个数组是一个已经有序的，他依然要去每轮遍历
- 适合小规模数据
